# 引入 Makefile 的原因与作用

    原因：在开发多文件项目时，手动使用 `gcc` 命令逐个编译源文件效率低下且容易出错。为此，我们引入了Makefile来自动化编译过程。

    Makefile 是一种构建脚本，它定义了项目的编译规则，包括源文件之间的依赖关系、编译命令等。它的主要优点包括：

        1. 自动识别依赖变化：Makefile 可以根据文件的时间戳判断哪些文件需要重新编译，从而避免重复编译未修改的文件，提高编译效率。
   
        2. 简化编译流程：只需输入 `make` 命令即可完成整个项目的构建，无需手动输入多个 `gcc` 命令。

        3. 便于维护和扩展：随着项目规模扩大，Makefile 可以灵活地管理多个源文件、头文件和目标文件之间的依赖关系。

        4. 统一构建标准：Makefile 提供了一种标准化的编译方式，使得不同开发者或持续集成系统可以一致地构建项目。

    因此，Makefile 在 C/C++ 项目开发中是一个非常重要的工具，极大地提升了开发效率和代码管理能力。
